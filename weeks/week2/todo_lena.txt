метод Get - получение списка дел
1) предусловие - есть хотя бы одно дело(id)
2) вызываем метод get 
 2.1) проверяем статус код - видим 200 (список получен)
 2.2) схема   {
    "id": "X",
    "text": "Y",
    "priority": Z
  }
 2.3) данные - получено столько элементов {}, сколько дел в списке


метод Delete
1) предусловие - есть дело(id)
2) запускаем метод delete по этому id
3) получаем статус код 204 - дело успешно удалено
4) запускаем метод по этому же айди - получаем код 404 - дело не найдено


метод Put
1) предусловие - есть дело(id)
2) вызываем put - получаем схему {
    "id": "X",
    "text": "Y",
    "priority": Z
  }
 2.1) в id копируем айди
 2.2) в request body меняем text на любой, priority = 5
     получаем 200 c измененными полями в response
3) передаем некое новое айди остальные поля те же, получаем 201 (создано новое дело)
4) проверяем поле priority на невалидность:
 4.1) вводим в priority букву алфавита, вызываем метод, получаем статус код 400 и в теле ответа ("'0xD0' is an invalid start of a value")
 4.2) вводим в priority цифру больше 10, вызываем метод, получаем код 400 и в теле ответа:
      "The field Priority must be between 1 and 10.")
 4.3) вводим в priority 0, вызываем метод, получаем статус код 400 и в теле ответа: "The field Priority must be between 1 and 10."
 4.4) отрицательное число
 4.5) нецелое число

метод Post
1) предусловие - есть id дела для добавления
2) в теле запроса вставляем:
 {
  "id": "наш айди",
  "text": "любой текст",
  "priority": цифра от 1 до 10
}

3) запускаем метод, получаем статус код 200(дело добавлено), в теле ответа то же что в теле запроса. Схема  {
    "id": "X",
    "text": "Y",
    "priority": Z
  }
4) запускаем метод с некорректными данными, то есть всё то же самое с полем Priority что описано выше для метода Put
получаем статус код 400 и в теле ответа  "The field Priority must be between 1 and 10."
5) запускаем метод с пустыми полями text и priority, получаем статус код 400
