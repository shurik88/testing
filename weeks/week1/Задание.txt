1.	Создать свою ветку с именем week1_имя (например, week1_andrew, week1_elena, week1_sonya) от ветки week1_andrew. Если будут проблемы с доступом, пишите
2. 	Установить fiddler  https://www.telerik.com/download/fiddler
3.	Найти и описать все api-запросы в системе (у api-запросов путь запроса будет начинаться всегда с "/api/").
		3.1		Сначала надо будет запустить систему через docker-compose up
		3.2 	Поскольку fiddler по умолчанию не показывает трафик на http, то необходимо обращаться к сайту не по localhost, а по имени машины, 
				3.2.1	Чтобы посмотреть имя машины введите в командной строке hostname
				3.2.2	Далее уже вводите в браузере http://имя_машины:5000, например, http://desktop-picehtg:5000
				3.2.3	В Fiddler должен отобразиться данный запрос
		3.3		Далее, используя fiddler, настройте фильтрацию, вам нужны только запросы на хост вашей машины и только api-запросы
		3.4		Переходите по всем вкладкам системы, найдите все запросы, их не так много :)
		3.5		Описать(в свободной форме в текстовом файле в папке weeks/week1/имя_исполнителя.txt, ) каждый найденный запрос
					Описание должно содержать:
						Метод запроса
						Схема или протокол запроса
						Путь запроса
						Строка запроса, если есть
						Статус код ответа
						Тело запроса (подробное описание json), если есть
						Тело ответа (подробное описание json), если есть
		3.6		Закоммитить изменение в файле и отправить изменения на сервер 
		(делайте по одному запросу и отправляйте на сервер, не забывайте получить изменения с сервера, там могут быть мои комментарии).
